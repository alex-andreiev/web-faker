Web Faker - Random Text Filler

Fill text fields and textareas with random test data, configurable Lorem Ipsum, names, emails, descriptions, chat messages, movie data, and Wikipedia summaries from keyboard commands.

Built for repeated testing work

Web Faker keeps placeholder data close to the field you are testing. Configure the shortcuts once, then fill active text fields directly from the keyboard.

Command mappings

Map short keys to names, emails, text snippets, Lorem Ipsum, chat messages, Wikipedia summaries, movie titles, or movie descriptions.

Linked movie data

Insert a movie title, then fill the matching description in the next field. Descriptions are enriched from TMDb overviews.

Lorem Ipsum generator

Generate classic Lorem Ipsum as words, sentences, or paragraphs, with separate settings for each mapped command.

Length ranges

Set per-dictionary minimum and maximum character ranges. Long text trims by sentence first, then by word when possible.

Lazy Wikipedia

Wikipedia is fetched only when the Wikipedia command is used, avoiding delays during normal local dictionary fills.

Double Key mode

Trigger mapped dictionaries by pressing the same key twice within a configurable delay, tuned to reduce accidental fills.

Website controls

Restrict the extension to specific hostnames when you want it active only on selected test environments.

Fast setup, predictable output

The extension is designed for QA sessions, admin panels, CMS testing, checkout flows, landing pages, onboarding flows, and prototypes where fields need plausible data.

1

Map commands

Assign keys like n, e, l, f, and p to dictionaries.

2

Set ranges

Define Lorem output per mapping and min/max lengths per dictionary. Use 0 when one side should be unlimited.

3

Type commands

Use the command character, command key, or Double Key mode inside active fields.

4

Keep context

Related movie title and description fields stay matched for one minute.

Install locally for development

1. Open chrome://extensions/
2. Enable Developer mode
3. Click Load unpacked
4. Select the Web Faker project folder
5. Reload after code changes

Private by default

Web Faker is designed for development and QA workflows where test data should stay local unless you explicitly use a network-backed dictionary.

No tracking

The extension does not include analytics scripts, advertising tags, or behavioral tracking.

No form collection

Form contents are not stored remotely or sent to a Web Faker server.

Local-first data

Most dictionaries are bundled locally. Wikipedia summaries are fetched only when the Wikipedia command is used.
